# Prompt Engineering

## 什么是 Prompt Engineering

Prompt Engineering（提示词工程）是通过设计输入提示词来引导 LLM 输出期望结果的技术。

## 基本原则

### 1. 清晰具体

```
❌ 写个文案
✅ 帮我写一篇小红书风格的奶茶推广文案，目标人群是大学生，语气活泼，200字左右
```

### 2. 提供上下文

给模型足够的背景信息，让它在正确的框架下回答。

### 3. 指定输出格式

明确告诉模型你想要的格式：Markdown、JSON、表格等。

## 常用技巧

### 角色设定（System Prompt）

```
你是一个资深 Python 工程师，擅长编写简洁高效的代码。
```

### 思维链（Chain-of-Thought）

引导模型逐步推理，而不是直接给答案：

```
让我们一步步思考：
1. 首先...
2. 然后...
3. 因此...
```

### Few-Shot（少样本示例）

在提示词中给出几个输入输出示例。

### 分步指令

将复杂任务拆解成多个步骤，分步完成。

## 高级技巧

| 技巧 | 说明 |
|------|------|
| **结构化提示** | 用 Markdown 标题分隔指令各部分 |
| **否定指令** | 明确说明不要做什么 |
| **Temperature 调节** | 低值（0-0.3）输出更确定，高值（0.7-1）更有创造力 |
| **约束 Token 数** | 用 max_tokens 控制输出长度 |

## 常见误区

- 提示词过于模糊
- 一次塞入太多指令
- 没有给出输出格式约束
- 期望模型知道你没有告诉它的信息
