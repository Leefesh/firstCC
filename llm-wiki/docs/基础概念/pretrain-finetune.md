# 预训练与微调

## 预训练（Pre-training）

在海量无标注数据上训练模型，学习通用的语言知识和能力。

- **数据**：互联网文本、书籍、论文等（TB 级别）
- **目标**：预测下一个 Token（Next Token Prediction）
- **产物**：基座模型（Base Model）
- **成本**：极高（数百万美元级）

## 微调（Fine-tuning）

在预训练模型的基础上，用特定数据进一步训练，让模型适应特定任务。

### 常见微调方式

| 方式 | 说明 | 数据需求 |
|------|------|----------|
| **全量微调** | 更新所有参数 | 较多 |
| **LoRA** | 只更新少量低秩矩阵，效果好 | 较少 |
| **QLoRA** | LoRA + 量化，消费级显卡也能跑 | 较少 |
| **SFT** | 监督微调，用人工标注的指令数据 | 中等 |

## 指令微调（Instruction Tuning）

用 `{指令, 回答}` 格式的数据微调，让模型学会遵循指令。

## RLHF（基于人类反馈的强化学习）

1. SFT 微调基座模型
2. 训练奖励模型（Reward Model）
3. 用强化学习（PPO）优化策略

ChatGPT 的成功关键之一就是 RLHF。

## 关系示意

```
互联网数据 → 预训练 → 基座模型 → 指令微调 → 助手模型
                                    ↓
                                RLHF → 对齐后的模型
```
